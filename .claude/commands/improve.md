# Improve Command

**Purpose**: Systematic code improvement with measurable enhancements

**Auto-Activations**:
- Persona: refactorer, performance, architect, qa (context-dependent)
- MCP: Sequential (analysis), Context7 (best practices), Magic (UI improvements)
- Flags: --validate, --loop (iterative improvement), --think

**Arguments**:
- `$ARGUMENTS` - Target for improvement
- `@<path>` - Specific path to improve
- `--focus <area>` - performance | quality | security | architecture | accessibility
- `--scope <level>` - file | module | project | system
- `--iterations <n>` - Number of improvement cycles (default: 3)
- `--interactive` - Confirm between iterations
